Nexus Hackathon (06.09.19 - 20.09.19)

Die Idee

  • Mehr Transparenz zum Recruiting Prozess
  • Messbare Ergebnisse zu Testaufgaben
  • Projektbezogene realistische Testfälle
  • Einfach erweiterbar
  • Kein Projekt- oder Framework bezogenes Wissen erforderlich

Fragen

  • Wie erreichen wir mehr Transparenz?
  • Wie prüfen wir Skills zur Code Erstellung und zum Debugging?
  • Wie bleibt es langfristig aktuell?
  • Wie kann es Spaß machen?

Testsuite

  • Definition unterschiedlichster Anwendungsfälle (z.B. Bugs fixen oder eigene Module entwickeln)
  • Erweiterung der Nexus internen Academy
  • Stetig wachsende Qualitfizierungsmöglichkeiten
  • Klare Aufgabenbeschreibung und Vorgaben (Branching)
  • Docker by default

Testsuite

  • Sichtbares Complexity Scoring via Annotations (1,2,4,8,16)
  • Automatisches pushen der Zwischenergebnisse nach Slack
  • Ranglist submitting nach Firebase
